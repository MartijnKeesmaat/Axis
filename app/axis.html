Axis

Axis is een component-driven boilerplate, dat uitgebreid kan worden tot een framework. Hiervoor is gebruik gemaakt van de BEM methodologie en een SMACCS geïnspireerde bestandsstructuur. Doormiddel van deze aanpak is het sneller makkelijker om een modulaire en schaalbare front-end omgeving op te zetten.

Boilerplates en frameworks

Het moeilijke van het maken van Axis was om te bepalen was het is. Het begon als een index.html bestand met een enkel css bestandje waarin wat code stond dat ik voor elk project gebruikte. Naarmate ik dit meer ging gebruiken, ging ik onderzoek naar boilerplates, optimale structuren en technieken voor benamingen. Dit resulteerde uiteindelijk in Axis.

Documentatie

Omdat het de bedoeling was om Axis te gebruiken in de teams waarin in werk is het belangrijk om een goede documentatie te schrijven. Dit is iets dat ik vaak opnieuw heb geschreven. Ik had vooral moeite met de grens te zoeken tussen te weinig en te veel te vertellen. Uiteindelijk heb ik een verzamelplek geschreven op Github waarbij de core concepten worden uitgelegd en hyperlinks zijn die doorlinken naar meer uitgebreide artikelen

Component system

Naast de core van Axis was het handig om een set van components te hebben. Hiervoor heb ik veel gekeken naar Atomic design van Brad Frost. Het belangrijkste was dat de components modular worden. Dit betekent dat alle onderdelen uitwisselbaar zijn met elkaar.

Resultaat

Het was erg leuk om Axis te werken, maar het was ook erg handig. Elke keer als ik een nieuw project heb opgezet —zowel als voor mezelf als voor bedrijven — heb ik Axis gebruikt. Dit zorgde altijd voor een overzichtelijke en schaalbare front-end omgeving wat mij veel tijd heeft gescheeld waardoor ik snel kon beginnen met coderen. Ook hebben anderen er profijt aan gehad om Axis te gebruiken, zowel medestudenten als collega’s hebben Axis met plezier gebruikt.
